# Clawso Labs 官网交互设定稿

## 项目定位

Clawso Labs 官网不是一个传统 AI 公司官网。

它要做成一个“沉浸式企业 AI 指挥舱入口”。

用户进入网站的第一感受不是“我在浏览一个网页”，而是：

> 我正在被邀请登上一艘企业 AI 指挥舰，进入 Clawso Labs 的 AI Command Deck，启动企业 AI 化进程。

Clawso Labs 的公司定位：

> Clawso Labs 帮助企业把零散 AI 工具，重构为可执行、可管理、可交付的 AI 生产系统。

Clawso 是 Clawso Labs 旗下产品，不等同于公司本身。

# 02. 首页开场：登舰流程

## 初始状态

用户打开网站后，不直接进入首页。

页面先进入黑屏启动状态。

背景：纯黑，带非常轻微的噪点、远处微弱星点、电子扫描线。

中间出现系统文字：

```text
CLAWSO LABS COMMAND SYSTEM
```

随后出现中文提示：

```text
检测到企业 AI 化请求
是否进入 Clawso Labs 指挥舱？
```

底部或中间出现两个按钮：

```text
登舰进入
稍后查看
```

按钮样式：

- 不是普通网页按钮
- 像航空器控制面板上的实体按钮
- 蓝青色发光描边
- 内部有轻微扫描线
- Hover 时边框扩散一圈光晕
- 点击时产生机械下压效果

## 初始文案建议

英文系统层：

```text
CLAWSO LABS COMMAND SYSTEM
ENTERPRISE AI TRANSFORMATION ACCESS
```

中文主提示：

```text
检测到企业 AI 化请求
是否进入 Clawso Labs 指挥舱？
```

按钮：

```text
登舰进入
稍后查看
```

# 03. 点击「登舰进入」后的启动动画

用户点击“登舰进入”后，进入 2.8 到 4 秒启动动画。

## 动画节奏

### 0.0s - 0.4s

按钮被按下。

画面轻微震动。

出现第一声机械反馈感：

```text
咔
```

视觉：按钮光线向外扩散，像启动电路被点亮。

### 0.4s - 1.0s

屏幕出现系统验证文字：

```text
身份确认中...
ENTERPRISE ACCESS VERIFIED
```

背景出现横向扫描线。

中心出现一个微弱圆形 AI Core 轮廓。

### 1.0s - 1.8s

五个系统依次上线：

```text
AI NATIVE ENGINE ONLINE
WORKFLOW CORE ONLINE
AGENT SYSTEM ONLINE
KNOWLEDGE RADAR ONLINE
RESULT DELIVERY SYSTEM ONLINE
```

每出现一行，对应一个模块灯光点亮。

节奏像飞船驾驶舱的开机序列：

```text
咔
咔咔
咔咔咔
```

### 1.8s - 2.6s

驾驶舱 UI 从黑暗中浮现。

- 顶部 HUD 框架展开
- 左右控制台滑入
- 中央 AI Core 亮起
- 底部实体按键区点亮
- 背景出现深色金属舱体边缘

### 2.6s - 3.2s

出现最终欢迎语：

```text
WELCOME TO CLAWSO LABS COMMAND DECK
```

随后进入正式首屏。

## 声音建议

可选，不强制。

声音不应吵闹，应该高级、克制。

建议声音：

- 低频引擎启动声
- 机械按钮轻响
- 系统上线提示音
- HUD 展开时的轻微电子声

移动端默认关闭声音，只做视觉反馈。

# 04. 首屏：Command Deck 企业 AI 指挥舱

## 首屏核心概念

首屏不是传统 Hero 区。

首屏应该是一个“可交互的企业 AI 指挥舱”。

用户进入后，看到一个未来航空器驾驶舱视角：

- 中央是发光 AI Core
- 周围是五个能力模块
- 顶部是系统状态栏
- 底部是驾驶舱按键区
- 背景是暗色未来企业舰桥
- 鼠标移动时 HUD 有轻微跟随

## 首屏主标题

推荐：

```text
启动企业 AI 化进程
```

备选：

```text
进入企业 AI Native 生产时代
```

```text
让企业拥有自己的 AI 指挥系统
```

## 首屏副标题

```text
Clawso Labs 帮助企业把零散 AI 工具，
重构为可执行、可管理、可交付的 AI 生产系统。
```

## CTA 按钮

```text
启动 AI 化诊断
查看能力控制台
```

按钮要有驾驶舱操作感：

- 主按钮：发光实体按钮
- 次按钮：透明 HUD 线框按钮
- Hover：边缘光效增强
- Click：轻微下压，触发短暂扫描反馈

# 06. 首屏交互规则

## 鼠标移动

- 背景 HUD 轻微跟随鼠标
- 星点或数据粒子有轻微视差
- 中央 AI Core 产生微弱方向偏移
- 左右面板有 3D 透视感，但不能晃得太厉害

## 鼠标悬停模块

当用户悬停任意模块：

1. 模块按钮边框增强发光
2. 状态灯闪烁一次
3. 中央 AI Core 切换到该模块的主题视觉
4. 主标题下方出现对应解释文案
5. 底部控制按钮同步高亮

## 点击模块

点击模块后，不直接跳普通锚点。

要像系统进入对应舱室：

```text
WORKFLOW CORE LOADING...
```

然后页面平滑滚动到对应 Section。

滚动过程可加入：

- HUD 纵向扫描
- 线条收束
- 模块放大后转场
- 背景暗场切换

## 滚轮下滑

第一次滚轮下滑时，从 Command Deck 进入 Mission Scan。

转场文案可短暂出现：

```text
MISSION SCAN INITIATED
```

中文：

```text
正在扫描企业 AI 化阻塞点
```

# 08. Section 02：Mission Scan 企业 AI 化扫描

## 视觉概念

进入第二屏时，像驾驶舱系统开始扫描一家公司。

画面中央可以是一个抽象企业系统地图：

- 部门节点
- 文档节点
- 流程节点
- 客户节点
- 数据节点

雷达线扫过后，暴露几个红色风险点。

## 标题

```text
大多数企业不是不会用 AI，
而是不知道如何把 AI 放进业务系统。
```

## 痛点模块

```text
AI 工具孤岛
员工各用各的，无法沉淀为组织能力。
```

```text
流程无法执行
AI 只能回答问题，不能真正进入业务链路。
```

```text
知识散落各处
文档、SOP、经验、客户资料无法被统一调用。
```

```text
结果不可管理
Demo 看起来很强，但上线后缺少权限、审计和责任边界。
```

```text
老板不知道从哪开始
想做 AI 化，但不知道先做哪个场景最值钱。
```

## 动效

- 每个痛点被雷达扫到后亮起
- 风险点先红色闪烁，再转为蓝色待解决状态
- 页面底部出现提示：

```text
SCAN COMPLETE / READY FOR TRANSFORMATION
```

# 09. Section 03：Capability Console 能力控制台

## 标题

```text
我们不是给企业加一个 AI 功能，
而是重构一套 AI 工作方式。
```

## 五大能力

### 01 AI 战略诊断

```text
从业务价值出发，判断企业最应该先 AI 化的场景，避免盲目投入。
```

### 02 Workflow 设计

```text
把重复工作、跨部门协作、内容生产、客户跟进和内部流程，设计成 AI 可执行的工作流。
```

### 03 Agent 系统搭建

```text
构建能理解上下文、调用工具、执行任务、交付结果的企业智能体。
```

### 04 企业知识智能化

```text
把企业文档、SOP、案例、销售话术和行业经验，变成可被 AI 调用的知识资产。
```

### 05 结果交付与运营

```text
不止做 Demo，而是交付可运行、可迭代、可度量的 AI 生产系统。
```

## 交互

五个能力模块像控制台按钮。

点击某个能力：

- 该模块展开
- 中央显示该能力的流程图
- 右侧显示交付物

例如点击 Workflow 设计：

交付物显示：

```text
业务流程拆解图
AI Workflow 蓝图
人机协作节点设计
工具调用清单
上线实施路线
```

# 14. 移动端适配规则

移动端不应该复刻复杂驾驶舱全景。

移动端应该像“掌上 AI 控制器”。

## 移动端开场

仍然保留黑屏询问：

```text
是否进入 Clawso Labs 指挥舱？
```

按钮：

```text
登舰进入
```

但启动动画缩短到 1.5 到 2 秒。

## 移动端首屏

布局：

- 顶部品牌
- 中央 AI Core
- 主标题
- 副标题
- CTA
- 下方横向滑动能力按钮

五大能力模块不要环绕，改成横向滑动控制条：

```text
Strategy
Workflow
Agent
Knowledge
Delivery
```

点击后中央 AI Core 切换视觉。

## 移动端导航

不使用普通汉堡菜单。

可以设计成一个底部 Dock：

```text
Command
Scan
Console
Product
Contact
```

底部 Dock 像飞船控制器，但要简洁。

## 移动端性能

- 粒子减少
- 3D 视差减弱
- 启动动画缩短
- 卡片转场简化
- 保留核心沉浸感，不牺牲可读性

# 15. 动效优先级

## 必须做

1. 黑屏登舰确认
2. 点击登舰后的系统启动动画
3. 首屏 AI Core 呼吸光
4. 五大模块 hover / click 反馈
5. 滚动进入 Mission Scan 的扫描转场
6. 架构层级依次点亮
7. 移动端简化适配

## 可以做

1. 鼠标视差
2. 粒子流动
3. 背景金属舱体微动
4. 模块声音反馈
5. 页面切换时 HUD 扫描线

## 不建议一开始做

1. 过重 3D 模型
2. 大量 WebGL 场景
3. 复杂物理动画
4. 真实游戏式操作系统
5. 需要加载很久的视频背景

第一版要优先保证：

- 打开速度
- 首屏冲击
- 交互记忆点
- 企业客户能看懂
- 手机端不崩

# 16. 技术实现建议

## 推荐技术栈

- Next.js
- React
- Tailwind CSS
- Framer Motion
- GSAP
- Three.js / React Three Fiber 可选
- Lenis 平滑滚动可选

## 第一版实现策略

不要一开始做重型 3D。

可以用：

- CSS radial-gradient
- SVG HUD 线条
- Framer Motion 交互
- GSAP timeline 控制登舰动画
- Canvas 粒子背景
- 伪 3D 面板透视

先做出感觉，再决定是否加入 Three.js。

# 19. 给设计师的一句话

这个官网不要设计成“AI 公司介绍页”。

要设计成：

> 一个企业客户点击“登舰进入”后，进入 Clawso Labs 的企业 AI 指挥舱，在里面完成一次从问题扫描、能力理解、架构认知、产品了解，到预约诊断的沉浸式旅程。

# 20. 给前端的一句话

第一版不要追求真 3D。

用黑屏登舰、HUD 动效、AI Core、模块按钮、滚动转场、视差和发光控制台，把“像进入飞船驾驶舱”的感觉做出来。

最重要的是：

> 让用户点击“登舰进入”的那一刻，觉得这个公司和普通 AI 服务商不一样。
